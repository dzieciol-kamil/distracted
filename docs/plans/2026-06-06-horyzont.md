# Horyzont Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add sky (WorldEnvironment + ProceduralSkyMaterial) and a colour-tweening ground plane to the game scene, so the horizon looks like sky above and grass/concrete below, changing colour as the zone progresses.

**Architecture:** A `WorldEnvironment` node provides a static procedural sky (always blue with drifting clouds). A large flat `MeshInstance3D` ("GroundPlane") sits just below road level and shows the zone's ground colour. A dedicated `EnvironmentController` node script listens to `GameState.zone_changed` and Tweens the ground material's `albedo_color`.

**Tech Stack:** Godot 4.4, GDScript (typed), gl_compatibility renderer — only built-in materials, no custom shaders.

### Task 1: Create environment_controller.gd

**Files:**
- Create: `src/scripts/game/environment_controller.gd`

- [ ] **Step 1: Write the script**

```gdscript
extends Node

const ZONE_COLORS: Array[Color] = [
	Color("#4caf50"),  # VILLAGE — green grass
	Color("#388e3c"),  # SUBURB  — darker green
	Color("#78909c"),  # TOWN    — grey-green
	Color("#546e7a"),  # CITY    — concrete grey
]
const TWEEN_DURATION: float = 0.5

var _tween: Tween

@onready var _ground_material: StandardMaterial3D = (
	$"../GroundPlane" as MeshInstance3D
).get_surface_override_material(0)

func _ready() -> void:
	_ground_material.albedo_color = ZONE_COLORS[GameState.zone]
	GameState.zone_changed.connect(_on_zone_changed)

func _on_zone_changed(new_zone: GameState.ZoneIndex) -> void:
	if _tween:
		_tween.kill()
	var target: Color = ZONE_COLORS[mini(new_zone, ZONE_COLORS.size() - 1)]
	_tween = create_tween()
	_tween.tween_property(_ground_material, "albedo_color", target, TWEEN_DURATION)
```

Notes on the path `$"../GroundPlane"`:
- `EnvironmentController` is a direct child of `Game` (scene root).
- `..` resolves to `Game`, so `../GroundPlane` is `Game/GroundPlane` — the sibling node added in Task 2.

- [ ] **Step 2: Commit**

```bash
git add src/scripts/game/environment_controller.gd
git commit -m "feat(horyzont): environment_controller — zone→colour tween"
```
